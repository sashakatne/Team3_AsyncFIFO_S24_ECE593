Fixes applied:
  RTL:
    D-1. Routed synchronizer_r2w_inst into the write clock/reset domain:
         .clk(wclk), .rst_n(wrst).
    D-2. Routed synchronizer_w2r_inst into the read clock/reset domain:
         .clk(rclk), .rst_n(rrst).
    D-3. Compute wHalfFull and rHalfEmpty from next local pointer
         occupancy, matching the predictive timing already used by
         wFull and rEmpty.
    D-4. Documented the known fifo_mem X window on rData. The TB gates
         reads on (rinc && !rEmpty), so no scoreboard compare consumes
         the uninitialized memory window.

  Testbench:
    T-1. Replaced the single-mailbox, one-transaction driver path with
         independent write/read stimulus mailboxes.
    T-2. Driver now forks write and read threads on their own clock
         domains and drives at negedge so stimulus is stable by the DUT
         sampling edge.
    T-3. Added write_mon_cb to the interface and split the monitor into
         write/read observers.
    T-4. Replaced the old array scoreboard with a queue reference model
         that calls $error on mismatch and prints a final PASS/FAIL
         verdict.
    T-5. Fixed the reset wait to block until both active-low reset
         domains are deasserted.

  Coverage / run infrastructure:
    C-1. Added top-level VCD dump for artifact generation.
    C-2. Replaced the old mixed run.do flow with a single compile,
         vopt, vsim, coverage-save, vcover-report sequence.
    C-3. Cleaned the M3 functional covergroup by removing impossible
         reset high-to-low bins and clock/data crosses; added write/read
         request-vs-flag coverpoints.
    C-4. Guarded vdel with `if {[file exists work]}` so a fresh remote
         run has no missing-work Tcl warning.
