Fixes applied (see design.md "Fixes applied" for full rationale):
  DUT (async_fifo.sv):
    D-1. Routed both synchronizer instances to their DESTINATION
         clock and reset domain (was source on both). synchronizer_r2w
         now has .clk(wclk) .rst_n(wrst); synchronizer_w2r now has
         .clk(rclk) .rst_n(rrst). Without this fix the two sync flops
         lived in the wrong domain on real silicon (zero metastability
         protection); simulation was passing only because of
         deterministic event ordering.
    D-2. Deferred. The fifo_mem array's data_out is combinational on
         b_rptr, and the array is X at elaboration -- so rData is X
         until the first write reaches the active read slot. Questa's
         strict always_ff single-driver rule prevents adding an
         elaboration-time initialiser. The post-fix TB makes this
         benign by gating all read-side comparisons on (rinc &&
         !rEmpty); see design.md for the full rationale.

  TB (full class layer rewritten):
    T-1. Driver / monitor / scoreboard now use proper iteration: the
         pre-fix `for i<1` loops in each class's main() were replaced
         by long-running threaded consumers that process 500 cycles
         per side.
    T-2. Scoreboard now uses a SystemVerilog queue as the reference
         model, tracks error_count, escalates mismatches via $error,
         and prints an explicit *** PASSED *** / *** FAILED *** verdict
         in final_report() at end of simulation.
    T-3. Driver reset wait corrected from the inverted
         wait(drv_if.wrst || drv_if.rrst) / wait(!drv_if.wrst ||
         drv_if.rrst) sequence to wait(drv_if.wrst === 1 &&
         drv_if.rrst === 1) with signals parked at 0 during reset.
    T-4. Driver now forks drive_writes (negedge wclk) and drive_reads
         (negedge rclk) as independent threads instead of serialising
         them in a single drive() task.
    T-5. Monitor split into observe_writes (posedge wclk via the new
         write_mon_cb clocking block) and observe_reads (posedge rclk
         via the existing monitor_cb clocking block). Each pushes to
         its own scoreboard mailbox; the scoreboard process_writes
         and process_reads consumers run independently.

  Interface (aysnc_fifo_interface.sv, NEW clocking block):
    Added clocking write_mon_cb @(posedge wclk) with input winc,
    wData, wFull, wHalfFull. Necessary because program code (the
    `test` program) schedules in the Reactive region per IEEE
    1800-2017 §4.4, which is AFTER module-side NBAs fire on the same
    posedge. Without the clocking block the monitor sees the
    post-edge view of flag signals, undercounting boundary writes
    when wFull transitions 0->1. The clocking block's default #1step
    input skew captures the pre-edge view -- the same view the DUT's
    Active-region always_ff saw -- which is the only way to get
    consistent transaction counts.

  Top (async_fifo_top.sv):
    Added $dumpfile / $dumpvars in an initial block so the
    evidence-package waveform renderer has a VCD to parse.

  run.do:
    Wrapped `vdel -all` in `catch {}` and added explicit `vlib work`
    so a fresh checkout doesn't error. Added vopt with
    `+cover=sbfec+asynchronous_fifo(rtl).` to instrument code
    coverage on the DUT only. Switched the vsim line to
    `top_optimized -coverage`. Added `coverage save async_fifo.ucdb`
    and `vcover report` calls at the end. Existing vlog ordering was
    preserved.

Caveats:
  #1  The pre-fix run.do had a bare `vdel -all` (no catch wrapper) and
      no coverage instrumentation. We added `catch {vdel -all}`,
      explicit `vlib work`, vopt with `+cover=sbfec+asynchronous_fifo(rtl).`,
      `vsim top_optimized -coverage`, and coverage save/report calls.
      Existing vlog ordering was preserved.
  #2  `vcover report ... -cvg -details` prints `No matching coverage
      data found` because the TB defines no covergroups (only code
      coverage is instrumented). Not a defect; functional coverage
      appears in M4 onward.
  #3  The filename `aysnc_fifo_interface.sv` (sic) is preserved.
      Renaming would force changes to async_fifo_top.sv and the
      package include path with zero functional benefit.
  #4  D-2 (rData X-propagation window) is deferred to a later fix
      pass that is willing to change the fifo_mem port list. The
      post-fix TB makes it benign via (rinc && !rEmpty) gating.
  #5  No bug-injection flags enabled. The M2 RTL has no `ifdef`-gated
      bugs; those appear only in Post_M5.
