Purpose:
  Demonstrate that the M1 root async FIFO (Style #1: async_cmp + direction
  flop + N-bit Gray pointers) compiles cleanly and simulates correctly

  provenance; this run reflects the corrected RTL/TB.

Fixes applied (see design.md "Fixes applied" for full rationale):
  DUT (async_fifo2.sv):
    1. Declared aempty_n and afull_n as wires in fifo2 (was implicit nets;
       eliminated 2 vlog-2623 / vopt-2623 warnings).
    2. Gated fifomem write enable by `winc & ~wfull` so an overflow write
       attempt no longer clobbers mem[0].
    3. Routed binary pointers (rbin, wbin) to fifomem instead of the
       Gray-coded wptr/rptr. The Gray pointers still drive async_cmp.
    4. Added rrst_n to the reset path of {rempty,rempty2} so the read side
       initializes correctly even if the two reset domains aren't aligned.
    5. Removed the unreachable `posedge high` (where `high` is a constant
       1'b1) from async_cmp's sensitivity list; trimmed the dead else branch.
    6. Initialized the `direction` register to 1'b0 to avoid an 'X' on
       aempty_n/afull_n during the brief window before wrst_n applies.
  TB (async_fifo2_tb.sv):
    7. Drive winc/wdata at negedge wclk (with blocking assignments) so they
       are stable when the DUT samples at the following posedge -- removes
       the one-cycle NBA-evaluation lag that previously kept winc asserted
       for one extra cycle after wfull. Combined with DUT fix #2 this is
       belt-and-braces.
